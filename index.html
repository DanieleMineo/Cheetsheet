Semantic HTML

Why use Semantic HTML?

There are 3 main reasons for using a sematic HTML language:

  • Accessibility: Semantic HTML makes webpages accessible for mobile devices and for people with disabilities as well. This is because screen readers and browsers are able to interpret the code better.
  • SEO: It improves the website SEO, or Search Engine Optimization, which is the process of increasing the number of people that visit your webpage. With better SEO, search engines are better able to identify the content of your website and weight the most important content appropriately.
  • Easy to understand: Semantic HTML also makes the website’s source code easier to read for other web developers.

Summary Table tags

Tag Name Description
<header> Header It's usually used as a container for the navigation links to the page or, more generally, for an introductory content.
<nav> Nav Is used to define a block of navigation links such as menus and tables of contents.
<main> Main Is used to encapsulate the dominant content within a webpage. By using <main> as opposed to a <div> element, screen readers and web browsers are better able to identify that whatever is inside of the tag is the bulk of the content.
<footer> Footer The content at the bottom of the subject information is known as the footer, indicated by the <footer> element. The footer contains information such as: Contact information, Copyright information, Terms of use, Site Map, Reference to top of page links
<section> Section Defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.
<article> Article The <article> element holds content that makes sense on its own. <article> can hold content such as articles, blogs, comments, magazines, etc. An <article> tag would help someone using a screen reader understand where the article content (that might contain a combination of text, images, audio, etc.) begins and ends.
<aside> Aside The <aside> element is used to mark additional information that can enhance another element but isn’t required in order to understand the main content. This element can be used alongside other elements such as <article> or <section>. Some common uses of the <aside> element are for: Bibliographies, Endnotes, Comments, Pull quotes, Editorial sidebars, Additional information.
<figure> Figure Figure is an element used to encapsulate media such as an image, illustration, diagram, code snippet, etc, which is referenced in the main flow of the document.
<figcaption> Figcaption <figcaption> is an element used to describe the media in the <figure> tag. Usually, <figcaption> will go inside <figure>. This is different than using a <p> element to describe the content; if we decide to change the location of <figure>, the paragraph tag may get displaced from the figure while a <figcaption> will move with the figure. This is useful for grouping an image with a caption.
<audio> Audio The <audio> element is used to embed audio content into a document. Although not always necessary, it’s recommended that we state the type of audio as it helps the browser identify it more easily and determine if that type of audio file is supported by the browser.
<video> Video The <video> element is used to embed videos content into a document.
<embed> Embed The <embed> element can embed any media content including videos, audio files, and gifs from an external source. This means that websites that have an embed button have some form of media content that can be added to other websites. The <embed> tag is a self-closing tag, unlike the <video> element. Note that <embed> is a deprecated tag and other alternatives, such as <video>, <audio> and <img>, should be used in its place, but is being taught for legacy purposes. Embed can be used to add local files as well as media content straight from some other websites.

Summary Table Attributes

The following table refers to the attributes that can be assigned to elements: <audio>, <video> and <embed>. There are multiple attributes that refer to the aforementioned elements, I will limit myself to reporting the main ones.

Attribute Name Description
src Search Specifies the URL of the audio file. Example: <audio> <source src="iAmAnAudioFile.mp3" type="audio/mp3"></source></audio>. In this example, we created an <audio> element. Then we created a <source> element to encapsulate our audio link. In this case, iAmAnAudioFile.mp3 is our audio file.
type Type Is used for specified the type by using type and named what kind of audio it is.
controls Controls When added in, a play/pause button will be added onto the video along with volume control and a fullscreen option.
autoplay Autoplay The attribute which results in a video automatically playing as soon as the page is loaded.
loop Loop This attribute results in the video continuously playing on repeat.
